# Getting Started

Welcome to Zeropoint Protocol! This guide will help you get up and running with our platform.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+** and npm
- **Git** for version control
- **Access to tinybox hardware** (for full functionality)
- **ROCm 5.7+** installed on your system

## Quick Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/FlynnVIN10/Zeropoint-Protocol.git
   cd Zeropoint-Protocol
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure environment**
   ```bash
   cp config/production.env .env.local
   # Edit .env.local with your configuration
   ```

4. **Start development server**
   ```bash
   npm run dev
   ```

5. **Open your browser**
   Navigate to `http://localhost:3000`

## Configuration

### Environment Variables

Key configuration options in your `.env.local`:

```bash
# Scope Controls
MOCKS_DISABLED=1
NO_SYNTHETIC_DATA=1
REAL_COMPUTE_ONLY=1

# Hardware Attestation
HARDWARE_ATTESTATION_REQUIRED=1
ROCM_VERSION=5.7.0

# Data Governance
PII_SCANNING_ENABLED=1
LICENSE_VALIDATION_ENABLED=1
CHECKSUM_VERIFICATION_ENABLED=1
```

### tinygrad Integration

The platform automatically integrates with tinygrad when:
- ROCm is properly installed
- tinybox hardware is detected
- Environment variables are configured

## Troubleshooting

### Common Issues

1. **Mock Detection Failing**
   - Ensure `MOCKS_DISABLED=1` is set
   - Check for mock implementations in your code
   - Review CI/CD configuration

2. **ROCm Not Detected**
   - Verify ROCm installation
   - Check GPU drivers
   - Ensure tinybox hardware is connected

3. **PII Scanning Errors**
   - Verify dataset integrity
   - Check checksums
   - Review license compliance
